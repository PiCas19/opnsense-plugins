# OPNsense Monitoring Bridge

The **OPNsense Monitoring Bridge** is a secure, scalable solution for monitoring and managing OPNsense firewalls within a DMZ environment. Built with FastAPI, it provides real-time system metrics, firewall rule management, emergency IP blocking, and integration with monitoring tools like Nagios, PRTG, and Prometheus. The bridge supports notifications via email, Slack, Microsoft Teams, and webhooks, ensuring timely alerts for critical events.

This project is designed to operate in a DMZ, with strict security measures including JWT authentication, IP whitelisting, and SSL/TLS encryption. It is deployable via Docker, Docker Compose, or Kubernetes, making it suitable for both small and enterprise-grade deployments.

## Features
- **Real-time Monitoring**: Collects system stats (CPU, memory, disk, uptime) and firewall metrics from OPNsense via its API.
- **Emergency Response**: Supports immediate and bulk IP blocking with configurable thresholds and durations.
- **Notifications**: Sends alerts via email, Slack, Teams, and custom webhooks for critical events (e.g., high CPU usage, security risks).
- **Monitoring Integrations**: Compatible with Nagios, PRTG, Prometheus, and Grafana for comprehensive observability.
- **Secure Design**: Implements JWT authentication, IP whitelisting, and SSL/TLS for secure operation in DMZ environments.
- **Deployment Flexibility**: Supports Docker, Docker Compose, and Kubernetes deployments.

## Architecture
The OPNsense Monitoring Bridge is a FastAPI application that interfaces with the OPNsense firewall API to retrieve system and firewall data. Key components include:
- **API Endpoints**: Exposed at `/health`, `/monitoring/*`, `/emergency/*`, `/nagios/*`, and `/prtg/*` (see [API Documentation](docs/api/endpoints.md)).
- **Services**: `opnsense_client.py` for API interactions, `monitoring_service.py` for metrics, `notification_service.py` for alerts, and `cyber_defense_service.py` for emergency actions.
- **Middleware**: JWT-based authentication for secure endpoint access.
- **Configuration**: Managed via a `.env` file (see `.env.example`).
- **Logging**: Configurable logging to `logs/` with rotation (see `config/logging.yaml`).
- **Deployment**: Containerized with Docker and deployable to Kubernetes (see `k8s/`).

The application runs in a DMZ network (`172.16.216.0/24`) and communicates with internal networks (`192.168.216.0/24`) for notifications and monitoring.

## Prerequisites
- **OPNsense Firewall**: Version 23.x or later with API access enabled.
- **Python**: Version 3.9+ for local development or testing.
- **Docker**: Version 20.10+ and Docker Compose for containerized deployment.
- **Kubernetes**: Optional for cluster deployments.
- **Dependencies**: Listed in `requirements.txt` (e.g., FastAPI, python-jose, prometheus-client).
- **Certificates**: Self-signed or CA-issued SSL certificates for secure communication (generated by `scripts/setup-bridge.sh` if needed).

## Installation
1. **Clone the Repository**:
   ```bash
   git clone https://github.com/your-org/opnsense-monitoring-bridge.git
   cd opnsense-monitoring-bridge
   ```

2. **Set Up Environment Variables**:
   Copy `.env.example` to `.env` and configure the required variables (e.g., `OPNSENSE_HOST`, `JWT_SECRET_KEY`, `SMTP_USERNAME`).
   ```bash
   cp .env.example .env
   nano .env
   ```

3. **Generate Certificates** (if not provided):
   ```bash
   ./scripts/setup-bridge.sh
   ```

4. **Install Dependencies** (for local development):
   ```bash
   pip install -r requirements.txt
   ```

## Configuration
Configure the application by editing `.env` with your environment-specific values. Key variables include:
- `OPNSENSE_HOST`, `OPNSENSE_API_KEY`, `OPNSENSE_API_SECRET`: OPNsense API credentials.
- `BRIDGE_IP`, `BRIDGE_PORT`: Bridge API address and port (default: `172.16.216.10:8443`).
- `JWT_SECRET_KEY`: Secret for JWT authentication.
- `ALLOWED_IPS`: Comma-separated IP ranges for access control (e.g., `172.16.216.0/24,192.168.216.0/24`).
- `SMTP_*`: Email notification settings (e.g., `SMTP_SERVER=smtp.gmail.com`).
- `PROMETHEUS_ENABLED`, `PROMETHEUS_PORT`: Prometheus metrics configuration.

See [Configuration Guide](docs/deployment/configuration.md) for details.

## Usage
1. **Run Locally** (for development):
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port 8443 --ssl-keyfile certs/server.key --ssl-certfile certs/server.crt
   ```

2. **Test API Endpoints**:
   ```bash
   ./scripts/test-api.sh
   ```

3. **Emergency IP Blocking**:
   ```bash
   ./scripts/emergency-block.sh 1.2.3.4 "Suspicious activity"
   ```

4. **Monitor Health**:
   ```bash
   ./scripts/health-check.sh
   ```

Access the API at `https://<BRIDGE_IP>:<BRIDGE_PORT>` (e.g., `https://172.16.216.10:8443/health`).

## Testing
The project includes unit and integration tests in the `tests/` directory, using pytest. To run tests:
```bash
pytest -v tests/
```
Set `TEST_MODE=true` and `MOCK_OPNSENSE_API=true` in `.env` for mock API responses during testing. Fixtures are provided in `tests/fixtures/` for consistent test data.
