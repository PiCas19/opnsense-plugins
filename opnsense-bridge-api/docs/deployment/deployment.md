# Deployment Guide

This document provides instructions for deploying the OPNsense Monitoring Bridge using Docker, Docker Compose, or Kubernetes. The application is designed to run in a DMZ environment with secure configurations for monitoring and emergency response.

## Prerequisites
- **OPNsense Firewall**: Version 23.x or later with API access enabled.
- **Docker**: Version 20.10+ and Docker Compose for containerized deployment.
- **Kubernetes**: Optional for cluster deployments (kubectl, Helm recommended).
- **Certificates**: Self-signed or CA-issued SSL certificates (generated by `scripts/setup-bridge.sh` if needed).
- **Environment Configuration**: A configured `.env` file (see `.env.example`).

## Docker Deployment
1. **Clone the Repository**:
   ```bash
   git clone https://github.com/your-org/opnsense-monitoring-bridge.git
   cd opnsense-monitoring-bridge
   ```

2. **Configure Environment**:
   Copy `.env.example` to `.env` and set the required variables:
   ```bash
   cp .env.example .env
   nano .env
   ```
   Example settings:
   ```plaintext
   BRIDGE_IP=172.16.216.10
   BRIDGE_PORT=8443
   JWT_SECRET_KEY=your-super-secure-jwt-secret-key
   OPNSENSE_HOST=192.168.216.1
   OPNSENSE_API_KEY=your_api_key
   OPNSENSE_API_SECRET=your_api_secret
   ```

3. **Generate Certificates**:
   ```bash
   ./scripts/setup-bridge.sh
   ```

4. **Build and Run**:
   ```bash
   docker-compose up --build -d
   ```

5. **Verify Containers**:
   ```bash
   docker ps
   ```
   Ensure `opnsense-bridge` and `prometheus` containers are running.

## Kubernetes Deployment
1. **Prepare Secrets**:
   Create a Kubernetes secret for `.env` variables:
   ```bash
   kubectl create secret generic opnsense-bridge-secrets --from-env-file=.env -n opnsense-bridge
   ```

2. **Apply Manifests**:
   Ensure the `k8s/` directory contains `secret.yaml`, `configmap.yaml`, `deployment.yaml`, `service.yaml`, and `ingress.yaml`. Apply them:
   ```bash
   kubectl apply -f k8s/ -n opnsense-bridge
   ```

3. **Verify Pods**:
   ```bash
   kubectl -n opnsense-bridge get pods
   ```

4. **Access the Service**:
   If using an Ingress, configure your DNS to point to `BRIDGE_IP` (e.g., `172.16.216.10`).

## Post-Deployment Verification
1. **Test API Connectivity**:
   ```bash
   ./scripts/test-api.sh
   ```

2. **Check Health Endpoint**:
   ```bash
   curl -k https://172.16.216.10:8443/health
   ```

3. **Monitor Logs**:
   ```bash
   tail -f logs/bridge.log
   ```

4. **Run Health Check**:
   ```bash
   ./scripts/health-check.sh
   ```

## Troubleshooting
- **Container Fails to Start**: Check Docker logs:
  ```bash
  docker logs opnsense-bridge
  ```
- **API Errors**: Verify `OPNSENSE_HOST`, `OPNSENSE_API_KEY`, and `OPNSENSE_API_SECRET` in `.env`.
- **SSL Issues**: Ensure `SSL_CERT_PATH` and `SSL_KEY_PATH` point to valid certificates or set `SSL_VERIFY_PEER=false`.
- **Kubernetes Issues**: Check pod logs:
  ```bash
  kubectl -n opnsense-bridge logs <pod-name>
  ```
- **Prometheus Not Scraping**: Verify `prometheus.yml` and ensure `PROMETHEUS_ENABLED=true`.

For additional configuration, see [Configuration Guide](configuration.md).
